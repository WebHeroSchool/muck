## Объекты

## Создание объекта

Для создание объектов рекомендуется использовать фигурные скобки, заместо конструктора `new Object`.

**Хорошо**
```
var item = {}
```

**Плохо**
```
var item = new Object()
```

## Зарезервированные слова

Не стоит использовать *[Зарезервированные слова](#зарезервированные-слова-(список))* в качестве ключей объектов.

**Хорошо**
```
var item = {
  defaults: { element: 'word' },
  hidden: true;
};
```

**Плохо**
```
var item = {
  default: { element: 'word' },
  private: true;
};
```

## Ключевые слова

Не стоит использовать [ключевые слова](#ключевые-слова-(список)) или их измененный аналог. Вместо них используйте синонимы.

**Хорошо**
```
var item = {
  type: 'solid'
};
```

**Плохо**
```
var item = {
  class: 'solid'
};
```

## Создание обьекта на 3 и более элементов

При создании объектов и массивов, содержащих большое количество элементов, образуя строки, длиной более 20 символов, необходимо выполнять следующий условия условия:
* Открывающая скобка располагается на той же строке;
* Каждое свойство оформляется на новой строке;
* Пробел после двоеточия;
* Закрывающая скобка располагается на новой строке;

**Хорошо**
```
var item = {
  defaults: { element: 'word' },
  type: 'solid',
  hidden: true
};
```

**Плохо**
```
var item = { defaults: { element: 'word' }, type: 'solid', hidden: true };
```

## Именование функции

Имя функции, в отличие от переменной, должно быть глаголом, но все еще на английском языке. Для имен, состоящих из нескольких слов, используйте *[camelCase](#camelCase)*

**Хорошо**
```

```

**Плохо**
```
function izmenenieobjecta() = {};

```

## Передача функций

При передаче функций, как аргумент в другую функцию,стоит оформлять код так:

```
  alert(i + ': ' + item + ' (массив:' + schedule + ')');
});
```

## Отступы

## Горизонтальные отступы

Отступ при вложенности - 2 пробела на каждый уровень.

**Хорошо**
```
if (item < 100) {
  for (var i = 0, iMax = items.length; i < iMax; ++i) {}
  }
}
```

**Плохо**
```
if (item < 100) {
   for (var i = 0, iMax = items.length; i < iMax; ++i) {}
}
  }
```

## Вертикальные отступы

Между блоками, циклами, функциями и т.д. следует оставлять пустую строку. Избегайте блоков кода более 9 строк подряд.

**Хорошо**
```
var a;
var aMax = items.length;
for (a = 0; a < aMax, ++a) {}

function showA() {}
```

**Плохо
```
var a;

var aMax = items.length;
for (a = 0; a < aMax, ++a) {}
function showA() {}
```

## Пробелы

Используйте пробелы между параметрами и не используйте пробелы между именем функции и скобкой. При создании анонимной функции необходимо использовать пробел перед скобкой. Исопльзуйте пробелы вокруг оператора.

**Хорошо**
```
function edit(item, itemTwo) {}

function (item, itemTwo) {}

if (item < 200) {}
```

**Плохо**
```
function edit (item,itemTwo) {}

function(item, itemTwo) {}

if (item<200) {}
```

## Скобки

Открывающая фигурная скобка располагается на той же строке, а закрывающая- на новой.

**Хорошо**
```
function edit(item, itemTwo) {
  if (item < 100) {}
}
```

**Плохо**
```
function edit(item, itemTwo)
{
  if (item < 100) {}}
```

## Кавычки

Всегда в коде скрипта используйте одинарные кавычки (`''`), если не требуется иного. Двойные ковычки приемлимы только при необходимости использования апострофа (`'`) или одинарной кавычки для других целей.

**Хорошо**
```
var string = 'строка';
var word = "you're";
```

**Плохо**
```
var string = "строка";
```

## Комментарии

Для пояснения кода используются комментарии, которые не исполняются оператором.
* Однострочные комментарии начинаются с двойного слэша и пробела `// `
* Многострочные комментарии заключаются в `/* ` и `*/`. За символом начала комментария идет пробел. Символ конца комментария располагается на новой строке

**Хорошо**
```
/* Комментарий
   на несколько строк
*/

// Однострочный комментарий
```

**Плохо**
```
/*Комменатрий на несколько строк*/
//Однострочный комментарий
```

## camelCase

camelCase - практика использования написания имен переменных, функций, и т.д.  без пунктуации и пробелов с наличием одной заглавной буквы во втором слове, при отсутствии таковой у первого слова.

**Пример**
